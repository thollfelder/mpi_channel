Detailed Description

Header of PT2PT SPSC BUF Channel.

This PT2PT SPSC BUF channel implementation uses the buffered send mode of MPI. To bookmark the count of sent elements the sender process stores the current buffer size, decrements it for every sent element and increments it for every received acknowledgement message from the receiver. To assert a correct usage of the buffered send mode channel_alloc_pt2pt_spsc_buf() needs to append enough memory for the receiver and the sender. The sender process checks for incoming acknowledgment messages and sends the element. The receiver process checks for incoming elements, receives the element and sends an acknowledgment message.

◆ channel_peek_pt2pt_spsc_buf()

int channel_peek_pt2pt_spsc_buf ( MPI_Channel *  ch)

Peeks at the channel and signals if messages can be sent (sender process calls) or received (receiver process calls).

Parameters
[in]chPointer to a MPI_Channel of type PT2PT SPSC BUF.
Returns
If the sender process calls it returns the current number of elements which can be sent. If the receiver process calls it returns 1 if a message can be received and 0 if no message can be received. If an error occures it returns -1.
Note
Returns -1 if internal problems with MPI related functions happen.
Since MPI will only let you check if and not how many messages can be received (MPI_Iprobe()) this function cannot return the number of buffered messages to the receiver.
